Shibaiinu Engine 使い方ガイド

シナリオの書き方と機能の詳細ガイドです。

セットアップスクリプト

新しいシナリオを始めるときは、付属のシェルスクリプトを使うと一発で雛形を用意できます。手で index.html を編集する必要はありません。

使い方

# プロジェクトルートで実行
./shibaiinu/scripts/setup-scenario.sh <シナリオ名> [タイトル]

# 例
./shibaiinu/scripts/setup-scenario.sh myStory
./shibaiinu/scripts/setup-scenario.sh myStory "私の物語"

引数

引数説明
<シナリオ名>JS 識別子として使える名前（英字始まり、英数字とアンダースコアのみ）。ファイル名・変数名・URL パスに使われます。
[タイトル]シナリオ選択画面に表示されるタイトル。省略時はシナリオ名と同じになります。

実行されること

  1. アセットディレクトリを作成: shibaiinu/assets/users/<名前>/(audio/bgm, audio/se, images/backgrounds, images/person)
  2. 空のシナリオファイルを作成: shibaiinu/scenario/<名前>.js(MessageEvent 1つだけ入った最小サンプル)
  3. index.html に import 文を追加し、scenarios 配列に新しいエントリを追加

生成されるシナリオファイル

// shibaiinu/scenario/myStory.js
import { Scenario } from '../core/scenario.js';
import { MessageEvent, SelectionEvent, InputEvent, WaitEvent } from '../core/events/index.js';

export const myStory = new Scenario({
  "start": {
    event: new MessageEvent([
      "ここに最初のメッセージを書いてください。",
    ]),
    next: null
  }
});
生成後はこのファイルを編集していつもどおりイベントを書き足してください。アセットも shibaiinu/assets/users/<名前>/ 配下に置けば、登録時の assetsPath が自動でその場所を指します。
注意: 既に同名のアセットディレクトリ・シナリオファイル・import が存在する場合はスクリプトが中止して何も変更しません（上書き防止）。

シナリオの削除

追加したシナリオを取り除くには削除スクリプトを使います。デフォルトでは アセットは残します（実作業の素材を誤って消さないため）。

# シナリオファイルと index.html の登録だけ削除（アセットは残す）
./shibaiinu/scripts/delete-scenario.sh myStory

# アセットも一緒に削除（確認プロンプトあり）
./shibaiinu/scripts/delete-scenario.sh myStory --with-assets

# 確認なしで全削除
./shibaiinu/scripts/delete-scenario.sh myStory --with-assets -y

オプション

オプション説明
--with-assetsshibaiinu/assets/users/<名前>/ も削除する
-y, --yes確認プロンプトをスキップする
部分的な状態（例: シナリオファイルだけ消した後にアセットだけ残った）でも再実行できます。見つからない要素はスキップされ、見つかったものだけ削除します。

基本構造

シナリオファイルの作成

// shibaiinu/scenario/myStory.js
import { Scenario } from '../core/scenario.js';
import { MessageEvent, SelectionEvent } from '../core/events/index.js';

export const myStory = new Scenario({
  "start": {
    event: new MessageEvent([...]),
    next: "次のイベントID"
  },
  // ...
});

index.htmlでシナリオを登録

import { ShibaiinuEngine } from './shibaiinu/engine.js';
import { myStory } from './shibaiinu/scenario/myStory.js';

const scenarios = [
  {
    id: 'myStory',
    title: 'シナリオのタイトル',
    description: 'シナリオの説明文',
    scenario: myStory,
    assetsPath: 'shibaiinu/assets/my_story/'
  }
];

const engine = new ShibaiinuEngine('game-container', { scenarios });
engine.start();
シナリオ選択画面: 複数シナリオ登録時はタイトル画面で選択、単一シナリオ時は直接開始します。

MessageEvent（メッセージイベント）

テキストを順番に表示するイベントです。

基本的な使い方

new MessageEvent([
  "普通のテキスト",
  "クリックで次へ進む"
])

キャラクター付きメッセージ

new MessageEvent([
  { speaker: "ユイ", text: "こんにちは！" },
  { speaker: "ユイ", image: "yui.png", text: "私はユイだよ" }
])

プロパティ一覧

プロパティ説明例
text表示するテキスト（必須）"こんにちは"
speaker話者名"ユイ"
imageキャラ画像ファイル名"yui.png"
positionキャラ表示位置"left" / "center" / "right"
background背景画像ファイル名"park.jpg"
backgroundFit背景表示モード"cover" / "contain"
bgmBGMファイル名"title.mp3"
se効果音ファイル名"click.mp3"

キャラクター表示

基本操作

// 画像を表示（中央、デフォルト）
{ image: "yui.png", text: "登場！" }

// 画像を非表示
{ image: null, text: "退場した" }

// 前の状態を維持（imageを省略）
{ speaker: "ユイ", text: "まだいるよ" }

複数キャラクター表示

最大3人のキャラクターを同時に表示できます。

// 左に表示
{ image: "yui.png", position: "left", speaker: "ユイ", text: "左側に立つよ" }

// 右に表示（左のキャラはそのまま）
{ image: "taro.png", position: "right", speaker: "タロウ", text: "右側に登場" }

// 左のキャラだけ消す
{ image: null, position: "left", text: "ユイが退場した" }

// 全員消す
{ image: null, position: "all", text: "全員退場" }
位置説明
left画面左側
center画面中央（デフォルト）
right画面右側
all全位置（非表示時のみ）

登場/退場アニメーション

imageAnim でキャラの表示/非表示時のアニメーションを指定できます。方向 = キャラが通る辺（左から入る／左へ抜ける）。

// 左からスライドイン
{ image: "yui.png", position: "left", imageAnim: "slide-left", text: "颯爽と登場" }

// 右へスライドアウト
{ image: null, position: "left", imageAnim: "slide-right", text: "右へ去っていった" }

// フェード（明示）
{ image: "taro.png", imageAnim: "fade", text: "ふわっと現れる" }
値挙動
fadeフェードイン／フェードアウト
slide-left左辺から入る／左辺へ抜ける
slide-right右辺から入る／右辺へ抜ける
slide-top上から入る／上へ抜ける
slide-bottom下から入る／下へ抜ける
imageAnim 未指定時は従来通りのフェードになります。背景は backgroundAnim で同じ値が使えます。

背景・BGM・効果音

背景画像

// 背景を変更
{ background: "night.jpg", text: "夜になった" }

// 背景を非表示
{ background: null, text: "暗転..." }

// 表示モードを変更（cover / contain）
{ background: "screenshot.png", backgroundFit: "contain", text: "全体を表示" }

背景アニメーション

backgroundAnim で背景の切り替え/非表示時のアニメーションを指定できます。値は imageAnim と同じ。

// 左から流れ込んで切り替わる
{ background: "city.png", backgroundAnim: "slide-left", text: "街に到着した" }

// フェードアウトで暗転
{ background: null, backgroundAnim: "fade", text: "視界が暗くなる..." }

// 上からスライドダウンで降ってくる
{ background: "sky.png", backgroundAnim: "slide-top", text: "空が広がる" }
値挙動
fadeフェードイン／フェードアウト
slide-left左辺から入る／左辺へ抜ける
slide-right右辺から入る／右辺へ抜ける
slide-top上から入る／上へ抜ける
slide-bottom下から入る／下へ抜ける

BGMの再生

// BGM再生（ループ）
{ bgm: "title.mp3", text: "ゲーム開始" }

// BGMをフェードアウトして停止
{ bgm: null, text: "静かになった..." }

// 別のBGMに切り替え
{ bgm: "battle.mp3", text: "戦闘開始！" }

効果音の再生

// 効果音再生
{ se: "door.mp3", text: "ドアが開いた" }

// BGMと効果音を同時に
{ bgm: "battle.mp3", se: "sword.mp3", text: "戦闘開始！" }

SelectionEvent（選択肢イベント）

プレイヤーに選択肢を提示するイベント。最大4つまで。

基本的な使い方

"choice1": {
  event: new SelectionEvent(["はい", "いいえ", "考え中"]),
  next: { 0: "yes_event", 1: "no_event", 2: "think_event" }
}
キーボード操作: 上下キーで選択、Enterで決定できます。

選択肢の上に問題文を表示

prompt オプションで選択肢の上に問題文を表示できます。[color] などの装飾タグや {flagName} も使えます。

"quiz": {
  event: new SelectionEvent(
    ["4", "5", "6"],
    { prompt: "2 + 3 は？" }
  ),
  next: { 0: "wrong", 1: "correct", 2: "wrong" }
}

選択時にフラグを設定する

afterHandler を使うと、選択時にフラグを設定できます。

"choice": {
  event: new SelectionEvent(
    ["はい", "いいえ"],
    {
      afterHandler: (index, choice) => {
        // index: 選んだ番号 (0, 1, ...)
        // choice: 選んだテキスト ("はい" or "いいえ")
        if (index === 0) {
          myScenario.setFlag("agreed", true);
        }
      }
    }
  ),
  next: { 0: "yes_route", 1: "no_route" }
}
注意: シナリオ内で myScenario.setFlag() を使うには、ファクトリ関数パターンを使うか、exportした変数を参照する必要があります（高度な使い方参照）。

InputEvent（入力イベント）

プレイヤーにテキスト入力を求めるイベント。名前入力などに使用。

基本的な使い方

"ask_name": {
  event: new InputEvent("あなたの名前は？", "player_name"),
  next: "greet"
},
"greet": {
  event: new MessageEvent([
    "こんにちは、{player_name}さん！"
  ]),
  next: null
}

オプション

new InputEvent("名前を入力", "name", {
  placeholder: "例: 太郎",     // プレースホルダー
  defaultValue: "ゲスト",     // 初期値
  maxLength: 10                // 最大文字数
})
引数説明例
prompt表示するプロンプト（必須）"あなたの名前は？"
flagName入力値を保存するフラグ名（必須）"player_name"
options.placeholder入力欄のプレースホルダー"例: 太郎"
options.defaultValue入力欄の初期値"ゲスト"
options.maxLength最大文字数（デフォルト: 50）20
入力値は自動的にフラグに保存され、{フラグ名} でテキストに埋め込めます。

WaitEvent（間を開けるイベント）

指定したミリ秒だけ間を開けてから次のイベントへ自動進行します。クリックやEnterではスキップできません。

"before_battle": {
  event: new WaitEvent(1500),  // 1.5秒待つ
  next: "battle_start"
}
前のメッセージはそのまま画面に残ります。シーン転換や緊張感の演出に。

テキスト装飾タグ

タグ説明例
{フラグ名}フラグの値を埋め込み{player_name}
[color:色]...[/color]文字色変更[color:red]赤[/color]
[b]...[/b]太字[b]重要[/b]
[shake]...[/shake]震えるテキスト[shake]ガタガタ[/shake]
[br]改行一行目[br]二行目
[wait:N]タイプライターを N ミリ秒だけ間を開ける...[wait:500]ん？
[code]...[/code]コードブロック[code]let x = 1;[/code]
[inline]...[/inline]インラインコード[inline]変数名[/inline]

使用例

// フラグの値を埋め込み
// scenario.setFlag("name", "太郎") の場合
"こんにちは、{name}さん！"  // → "こんにちは、太郎さん！"

// 色変更
"[color:#ff69b4]ピンク色[/color]のテキスト"

// 太字
"[b]重要な言葉[/b]"

// 震えるテキスト
"[shake]ガタガタ...[/shake]"

// コードブロック（コピーボタン付き、タイプライターなし）
"[code]const x = 1;[br]console.log(x);[/code]"

// タイプライターを途中で止めて間を作る（500ms 待機）
"よし、いくぞ...[wait:500]覚悟しろ！"

// 組み合わせ
"[color:#ffd700][b]金色で太字[/b][/color]"

フラグ管理

ゲームの進行状況を記録し、条件分岐に使用できます。

フラグによる条件分岐

"door": {
  event: new MessageEvent(["扉がある..."]),
  next: (flags) => {
    if (flags.has_key) {
      return "open_door"
    }
    return "locked_door"
  }
}

SelectionEventでのフラグ分岐

"choice": {
  event: new SelectionEvent(["進む", "戻る"]),
  next: (flags, selectedIndex) => {
    if (selectedIndex === 0) {
      return flags.is_strong ? "win_battle" : "lose_battle"
    }
    return "go_back"
  }
}

フラグ関連メソッド

scenario.setFlag("key", true)     // フラグを設定
scenario.setFlag("count", 5)      // 数値も可
scenario.getFlag("key")           // フラグを取得
scenario.hasFlag("key")           // フラグが存在するか
scenario.clearFlags()             // 全フラグをクリア

高度な使い方

シナリオ内でフラグを設定する（ファクトリ関数パターン）

シナリオ内から自身を参照するには、ファクトリ関数を使います。

// shibaiinu/scenario/myStory.js
import { Scenario } from '../core/scenario.js';
import { MessageEvent, SelectionEvent } from '../core/events/index.js';

export function createMyStory() {
  const scenario = new Scenario({
    "start": {
      event: new MessageEvent(
        ["冒険の始まり..."],
        null,
        () => { scenario.setFlag("started", true) }
      ),
      next: "choice"
    },
    "choice": {
      event: new SelectionEvent(
        ["戦う", "逃げる"],
        {
          afterHandler: (index) => {
            if (index === 0) {
              scenario.setFlag("brave", true);
            }
          }
        }
      ),
      next: { 0: "battle", 1: "escape" }
    }
  });
  return scenario;
}

index.htmlでの呼び出し

import { createMyStory } from './shibaiinu/scenario/myStory.js';

const scenarios = [
  {
    id: 'myStory',
    title: 'タイトル',
    description: '説明',
    scenario: createMyStory()  // 関数を呼び出す
  }
];

ファイル配置

アセットルートの構成

shibaiinu/assets/ はシステム提供のものとユーザー作成のものを分けて管理します。

shibaiinu/assets/
├── system/              # エンジン同梱（編集しない）
│   ├── audio/se/        # select.mp3 / decide.mp3 などのシステム SE
│   ├── images/
│   └── sample_game/     # サンプルシナリオ用アセット
└── users/               # setup-scenario.sh で作成（編集する）
    └── <シナリオ名>/
        ├── audio/{bgm,se}/
        └── images/{backgrounds,person}/

シナリオ内のサブディレクトリ

各シナリオの assetsPath 配下には以下のサブディレクトリを置きます。シナリオから image: "yui.png" のようにファイル名だけ指定すれば、対応するフォルダから読み込まれます。

種類配置場所（assetsPath 配下）
キャラ画像images/person/
背景画像images/backgrounds/
BGMaudio/bgm/
効果音audio/se/
シナリオ登録時に assetsPath を指定すると、シナリオごとに異なるアセットフォルダを使用できます。setup-scenario.sh で作成した場合は shibaiinu/assets/users/<名前>/ が自動で割り当てられます。

シナリオ例

export const sampleScenario = new Scenario({
  "start": {
    event: new MessageEvent([
      { background: "room.jpg", bgm: "morning.mp3", text: "朝、目が覚めた。" },
      { speaker: "母", image: "mom.png", text: "おはよう！" }
    ]),
    next: "choice"
  },
  "choice": {
    event: new SelectionEvent(["食べる", "食べない"]),
    next: { 0: "eat", 1: "skip" }
  },
  "eat": {
    event: new MessageEvent(["朝ごはんを食べた。"]),
    next: null
  },
  "skip": {
    event: new MessageEvent(["お腹が空いてきた..."]),
    next: null
  }
});
